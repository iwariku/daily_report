 ##  わかったこと

- 並行処理とは、複数の処理を切り替えながら進めることで、同時に実行しているように見せる仕組み。

- `Golang`変数宣言した変数は必ず使わなければならない。

- 返り値のところで、`result`と記述してあるので、`price * 2`した結果を処理内に記述しない方法もある。

```
func Double(price int) (result int) {
	result = price * 2
	return
}
```

- `ジェネレーター`とは、なんらかのルールに従って連続した値を返し続ける仕組みのこと。

- `switch文のcase`で、列挙型の式と演算子を使った式を混在させるとエラーになる。

- 繰り返し文の時に、ラベルをつけることで、繰り返し文で抜け出す際にラベルで指定した位置に飛ぶことができる。

- `defer`は、関数の終了時に実行される処理を登録することができる。

- `defer`で登録された処理は、他にある処理がすべて実行されてから、実行される。

- `defer`で登録された処理は、最後に登録された順に実行される。

- `go`と記述することで、並行処理を走らせることができる。

- `init`は、`main関数`よりも先に実行される、初期化処理のこと。

- `map`の記述方法。<br>`map[key]value{}` (例) `m2 := map[string]int{"A": 100, "B": 200}`

- 複数の返り値がある時、`_`を使うことで、対応する値を使わないようにできる。

- `チャネル`とは、複数のgotoutine間でのデータの受け渡しをするために設計されたデータ構造。

- ポインタとは、値型に分類されるデータ構造(基本型、参照型、構造体)のメモリ上のアドレスとと型の情報のことです。値の情報を持つのではなく、値が置いてある場所(アドレス)を値として持つ変数。

- ポインタの`*`は、参照先の値に直接アクセスするための操作。
