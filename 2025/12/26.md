 ##  わかったこと

- `uesEffect`とは、コンポーネントの状態や`props`の変化、または初回秒がをトリガーに、副作用となる処理(API通信やlocalStorage、その他多数)を実行する仕組み。

- `useEffect`内で`async`と使う時は
    ```
    const Home = () => {
      useEffect(async () => {
        const getPosts = () => {};
      }, []);
    ```
    と書くのではなく
    ```
    const Home = () => {
      useEffect(() => {
        const getPosts = async () => {};
      }, []);
    ```
    と新しく`async関数`を定義する。

- `window.location.href="/"`で指定したpathへリダイレクトできる。

- Reactのリファクタリングでは、`state`に重複がないか？宣言したstateの組み合わせの初期値に整合性が取れているかなども見る。

- 宣言的プログラミングとは、UIを細かく管理する(命令型)のではなく、視覚状態ごとにUIを記述する。

- コンポーネントを開発するとき: 
    1.コンポーネントの視覚状態をすべて特定する。
    2.状態を変更するための人間及びコンピューターのトリガを決定する。
    3.`useState`でstateをモデル化する。
    4.バグや矛盾を避けるため、不必要なstateを削除する。
    5.stateを設定するためのイベントハンドラを定義する。

- 2つのstate変数が一緒に更新される場合は、それらを1つにまとめることを検討する。
