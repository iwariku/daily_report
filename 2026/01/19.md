 ##  わかったこと

- `interface`で宣言した関数を実装しているコードがあれば、その型は`interface`を満たしている。
    ```
    // interfaceの宣言
    type Greeter interface {
    	Greet()
    }
    
    type Human struct{}
    
    // interfaceを満たす
    func (h Human) Greet() {
    	fmt.Println("こんにちは")
    }
    
    ```

- `interface`は「形(型)」しか保証しない。<br>「正しい処理内容」や「意味の正しさ」までは保証しない。

- `Context`型の主な役割
    - 処理の締め切りを伝達
    - キャンセル信号の伝播
    - リクエストスコープ値の伝播

- goroutine = Goランタイムが並列実行する関数(仕事の単位)

- goroutineの数 = 並行に実行したい「仕事」の数

- contextとは、「異なるgoroutine上で情報伝達するための機構」
