 ##  わかったこと

- **DBとアプリケーションの考え方**

|レイヤー | 役割(考え方) |
| ---- | ---- |
| DB(データ構造) | **「事実の倉庫」** 重複を避け、整合性を保ち、データを安全に保存することに特化する |
| アプリケーションレイヤー | **「体験の窓口」** 画面に何を表示するか、どの情報を隠すか、どんな形式が扱いやすいかを優先する |

- **DBのニックネームとASの役割**

| 項目 | 記述例 | 役割 | 目的 |
| ---- | ---- | ---- | ---- |
| テーブルの別名 | `FROM tweet t` | テーブルにニックネームをつける | クエリを短く書き、カラムの所属を明確にする |
| カラムの別名 | `COUNT(*) AS count` | 列に新しい見出しをつける | DBのカラム名、アプリケーション側で受け取る時の名前をつける |
