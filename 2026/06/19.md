##  わかったこと

- IPアドレス`10.0.0.1`はネットワーク部の`10.0.0.`とホスト部の`1`に分けられる

- `CIDR`とは、ネットワークのアドレスとネットワーク部の範囲を表す記法。利用できるIPアドレス範囲がわかる

- パブリックサブネット
  - インターネットに**直接アクセスできる**サブネット
  - サーバーは、グローバルIPアドレスを持つことで外部からアクセスできる。

- プライベートサブネット
  - インターネットに**直接アクセスできない**サブネット

- セキュリティグループ
  - VPC内のサーバーなどに紐づけて通信を制御するための**仮想ファイヤーウォール**
  - アウトバウンド(出ていく通信)とインバウンド(入ってくる通信)のルールを定義してアクセスを管理する。

- アウトバウンドで外に出るのはOK。
    その通信に対するレスポンスは、インバウンドルールを書いていなくても戻って来れる。

- セキュリティグループの特徴
  - ホワイトリスト方式：インバウンド・アウトバウンドルールに記載のないものは全て不許可
  - 高い柔軟性：許可する対象のIPアドレスの他、特定のセキュリティグループを持つリソースで指定も可能
  - 即時反映：ルールの変更は全ての関連するインスタンスに即時反映。

- ローリングアップデートとは、複数あるコンテナを1つづつ落として、巻き替えてを順番で行い、バージョンアップしたコンテナを全て立ち上げる。

- Blue/Greenデプロイは、**新旧のバージョンを並行稼働させる**ことで、ダウンタイムを最小限に抑え、リスクを管理しやすくする手法。

|  | Blue/Greenデプロイ | ローリングアップデート|
| ---- | ---- | ---- |
| GOOD | ・ダウンタイムの最小化<br>・迅速なロールバック<br>・安全なテスト環境 | ・段階的なデプロイ<br>・リソースの効率的な利用 |
| BAD | ・コストの増加<br>・管理の複雑さ | ・ロールバックの複雑さ |
