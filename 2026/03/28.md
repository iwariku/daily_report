### Next.js
#### Pages Routerのレンダリング戦略
- CSR(クライアントサイドレンダリング): クライアント側でHTMLを構築する
- SSR(サーバーサイドレンダリング): リクエスト毎に、サーバーでHTMLを構築する
- SSG(静的サイト生成): ビルド時点でHTMLを構築する
- ISR: ビルド時点でHTMLで構築し、指定した間隔でHTMLを再構築

#### Server ComponentとClient Component
- レンダリング場所
    - Server Component
        - サーバー
    - Client Component
        - サーバー、クライアント

- 扱える関数、パラメータ
    - Server Component
        - Next.jsで用意されているDynamic API(cookie, headers, searchParams, ...)、ライブラリ(Prisma)が使える
    - Client Component
        - ブラウザAPI(document, localStorage...)、イベントハンドラー(onClick, onChange...)、React hooks (useState, useEffect...)が使える
