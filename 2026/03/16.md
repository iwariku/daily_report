##  わかったこと
### TypeScript
#### Utility Types
-  `Partial<T>` `Type Draft = Partial<User>;`<br>全てのプロパティを「オプショナル(任意)」に変換する。入力フォームや下書きなどで使う。
-  `Required<T>` `Type Full =Required<User>;`<br>全てのプロパティから「?」を取り除き、「必須」に変換する。入力完了後のデータ定義に使える。
- `Readonly<T>` `Type Locked = Readonly<User>;`<br>全てのプロパティに「readonly」を付与し、書き換え不可にする。不変オブジェクトを作るのに必須。
-  `Pick<T, K>` `type Item = Pick<User, "id" | "name" >`<br>指定したキーだけを「抽出」して新しい型を作ります。必要な情報だけを切り出すのに使います。
- `Omit<T, K>` `type Public = Omit<User, "password">`<br>指定したキーを「除外」して、残りのプロパティで型を作ります。不要な情報を隠すのに便利です。
- `Record<T, K>` `type Scores = Record<string, number>`<br>キー(K)と値(T)の型を指定して、マップ(辞書)型を作ります。インデックスシグネチャの代わりになります。

#### Promise<T>非同期処理の型定義
- `async`関数は、値を直接返すのではなく、必ずPromiseでラップして返します。
    ```
    // Promiseで返却する時にWeathreData型のPromise型で返却する。
    const fetchWeather = async (): Promise<WeatherData> => {
      await new Promise((resolve) => setTimeout(resolve, 1000));

      return {
        temperature: 25,
        condition: 'Sunny',
      };
    };
    ```
